# 区级行政区天气地图 — 可行性分析

> 来源：2026-06-20 米豆需求
> 目标：Android 原生界面，展示当前区县 + 相邻区县天气，可直观感知周边下雨情况

---

## 一、核心流程

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ GPS定位  │ ──→ │ 逆地理编码 │ ──→ │ 行政区查询 │ ──→ │ 批量天气  │
│         │     │ (得adcode)│     │ (得所有区)│     │ (每区天气)│
└─────────┘     └──────────┘     └──────────┘     └──────────┘
                                                         │
                                              ┌──────────┘
                                              ▼
                    ┌──────────────────────────────────┐
                    │  Android MapView + Polygon 渲染   │
                    │  · 区县边界（polyline→Polygon）   │
                    │  · 天气填充色（雨=蓝、晴=金）     │
                    │  · 温度/天气图标 Marker           │
                    │  · 点击区县卡片展开详情           │
                    └──────────────────────────────────┘
```

---

## 二、数据链路

### Step 1: 定位 → 区县

**逆地理编码**：`/v3/geocode/regeo?location=lng,lat`
→ 返回 `adcode`（如 `220102` = 南关区）+ 区县名

消耗：1次/refresh（基础LBS配额，150,000次/月 → 无限够）

### Step 2: 获取全市区县列表

**行政区 API**：`/v3/config/district?keywords=长春市&subdistrict=1&extensions=all`
→ 返回所有区县：名称、adcode、边界坐标(polyline)、中心点

消耗：1次/refresh（基础LBS配额）

### Step 3: 批量查询各区天气

**天气 API**：`/v3/weather/weatherInfo?city=220102&extensions=base`
× 每个区县 1 次

消耗：N次/refresh（天气配额，5,000次/月）

### Step 4: 地图渲染

**Android Map SDK**：
- `Polygon` 绘制各区边界 + 填充天气主题色
- `Marker` 标注温度 + 天气图标
- `InfoWindow` 点击展开详情卡片

---

## 三、配额分析

### 按城市规模（以长春为例，约10个区县）

| 操作 | 次/refresh | API 组 | 月可用 | 月可刷次数 |
|------|-----------|--------|--------|-----------|
| 逆地理编码 | 1 | 基础LBS | 150,000 | ∞ |
| 行政区查询 | 1 | 基础LBS | 150,000 | ∞ |
| 天气查询 | ~10 | 天气预报 | 5,000 | ~16次/天 |
| **合计** | **~12** | — | — | **够用** |

### 北京/上海（16个区）

| 操作 | 次/refresh |
|------|-----------|
| 天气查询 | ~16 |
| 月可刷次数 | ~10次/天 |

**结论**：大城市每天可刷新 10-16 次，手动刷新完全够用。如需自动轮询需升级配额。

## 五、对比：高德天气 vs 和风天气做同一功能

| 维度 | 高德方案 | 和风方案 |
|------|---------|---------|
| 行政区边界 | ✅ 内置 polyline，一步到位 | ❌ 无，需自备 GeoJSON |
| 天气数据丰富度 | ⚠️ 仅6字段，无小时级/AQI/降水mm | ✅ 全维度 |
| 地图渲染 | ✅ Android SDK 原生支持 | ❌ 需自行接入地图SDK |
| 免费额度 | 5,000次/月(天气) | 30,000次/月(1,000/天) |
| 一站式程度 | 🏆 数据+地图+边界一家解决 | 需拼装多个服务 |

**推荐架构**：**高德负责空间数据（边界+地图渲染），和风负责天气数据（内容丰富度）。**

```
高德 SDK → 地图底图 + 区县边界 + Polygon渲染
高德 API → 逆地理 + 行政区查询（获取边界）
和风 API → 每区天气数据（温度/降水/AQI/预警）
```

## 六、需要和风提供的额外数据

高德天气过于基础，天气地图需要以下和风数据增强：

1. **降水量(mm)** → 雨天时在区县上叠加雨量数字
2. **降水概率** → 未下雨区县的降水预判
3. **天气预警** → 预警区县红色闪烁边框
4. **小时级预报** → 时间轴切换查看未来几小时
5. **AQI** → 污染区县棕色覆盖

## 八、结论

✅ **技术上完全可行。** 高德提供了关键的数据基础设施（行政区边界 polyline + 逆地理 + Android Map SDK），一行代码搞定"在哪"和"边界在哪"。

⚠️ **天气数据需要和风补充。** 高德天气只有最基础的 6 字段（温度/风/湿度/描述），没有降水量、没有 AQI、没有预警——这些是"感知下雨"场景的核心。

🎯 **最佳策略**：高德做地图渲染基础设施（全栈覆盖），和风做气象数据引擎。两者互补，不存在竞争。
