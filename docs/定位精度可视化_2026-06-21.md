# 定位精度可视化 — 2026-06-21

## 目标
在天气APP中增加**定位溯源 + 每源空间分辨率可视化**，让用户看到：
1. 定位来源（GPS/IP/手动/兜底）
2. 定位精度（±Xm 或 ±Xkm）
3. 每个数据源的空间精度（~3km网格 / 区级 / 地级市）

## 改动文件

### 1. 类型定义 `src/types/weather.ts`
- **新增** `LocationSource = "gps" | "ip" | "manual" | "fallback"`
- **新增** `SpatialPrecision = "point" | "district" | "city"`
- `GeoLocation` 增加字段：
  - `province: string` — 省级行政区
  - `city: string` — 地级市名（CMA place参数用）
  - `precision: LocationSource` — 定位溯源
  - `accuracyMeters: number | null` — 估算精度
- `SourceState` 增加字段：
  - `spatialPrecision: SpatialPrecision`
  - `spatialPrecisionLabel: string` — 人类可读（"~3km网格"等）
- `SourceId` 从 `"seniverse"` 改为 `"cma"`

### 2. 定位服务 `src/lib/location.ts` (全新)
- **GPS优先**: Capacitor Geolocation (高精度模式, timeout 10s)
- **IP回退**: 高德 IP 定位，从rectangle计算精度半径
- **最后兜底**: 长春硬编码坐标 (fallback, ±50km)
- 逆地理编码 → 街道+区+市+省
- 所有写入 Zustand Store

### 3. 数据源注册 `src/lib/weather-service.ts`
- 每个AdapterEntry增加 spatialPrecision/spatialPrecisionLabel
- 源精度映射：
  - 和风天气: point / ~3km网格
  - Open-Meteo: point / ~11km网格
  - 高德地图: district / 区级
  - 彩云天气: point / ~5km网格
  - 中国气象局: **city / 地级市**（见下方CMA调查）

### 4. 首页 `src/pages/HomePage.tsx`
- **新组件 `SourcePrecisionBar`**: 在LocationBar和Hero之间，显示每源的空间精度标签
  - point → 绿色边框
  - district → 蓝色边框
  - city → 琥珀色边框
- **LocationBar 升级**: 定位来源徽章（GPS/IP/手动/默认位置）+ 精度距离

### 5. Mock数据 `src/lib/mock-data.ts`
- `sourceIds` 从 `seniverse` → `cma`
- `mockSources` 增加 spatialPrecision/spatialPrecisionLabel

### 6. Store `src/lib/store.ts`
- `loadMock()` 中 GeoLocation 增加 province/city/precision/accuracyMeters

## CMA 精度调查结论

| place参数 | 结果 | 备注 |
|-----------|------|------|
| 昌邑区 | NOT FOUND | CMA不支持区级 |
| 船营 | NOT FOUND | 同上 |
| 龙潭 | NOT FOUND | 同上 |
| 丰满 | NOT FOUND | 同上 |
| 吉林 | shi=舒兰 (126.93,44.39) | 偏北50km |
| 昌邑 | shi=昌邑 (119.42,36.86) | 山东昌邑市，距离900km |
| 长春 | shi=长春 (125.23,43.89) | 正确，但非目标城市 |

**结论**: CMA的apihz接口只覆盖县级市/县，不覆盖地级市的区。昌邑区无法通过CMA获取区级数据，精度固定在"地级市"级别（实际解析到最近的县级市舒兰）。
