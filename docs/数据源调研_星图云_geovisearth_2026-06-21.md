## 二、API能力

### 2.1 已知端点

| 端点 | 路径 | 说明 |
|------|------|------|
| 坐标批量天气 | `/v2/weather/batch/point?location=lon,lat&token=` | 实况+48h逐小时+预警 |
| 区划天气 | `/v2/cn/area/basic?location=WTX_CH...&token=` | 按areaCode查询 |

**统计：22个独立实况字段，是现有五源中最多的。**

### 2.3 逐小时预报 (fcHourData)

- 时长：48小时
- 每时次14参数：fc_time, wp_code, wp, tem, real_tem, dp_tem, ws, ws_desc, wd, wd_desc, rh, prs, vis, cloud_cover, pre

### 2.4 预警 (alertData)

JSON格式，含：alertId, typeCode, type(霜冻/暴雨等12类), level(Yellow/Red/Orange), title, detail

### 2.5 定位

返回 areaCode(WTX_CH...) + path("省,市,县")，区县级精度

## 三、RainPulse 降水临近预报

| 参数 | 值 |
|------|-----|
| 更新频率 | **6分钟** |
| 预报时长 | 未来2小时 |
| 空间分辨率 | **1km (街道级)** |
| 空间范围 | 中国区(73~135.5°E, 12.15~54.2°N) |
| 算法 | 深度学习雷达回波预测+反演降水 |
| 数据格式 | PNG/WebP图像层 |
| 产品要素 | 回波强度、降水强度 |

**局限**：RainPulse返回的是图像层，不是JSON数值，需要地图叠加渲染，不能直接用于数据融合。

## 六、技术优势

1. **数据模型独立性**：卫星+地面观测+AI融合，不同于CMA/GFS/ECMWF任一
2. **字段编号化**：全数值字段，方向用角度、风速用m/s、趋势用Steady/Rising/Falling
3. **响应速度**：声称<30ms，远超同行
4. **上市公司背景**：数据质量和持续运营保障
