## 3. 天气查询 API `/v3/weather/weatherInfo`

### 3.1 请求参数

| 参数 | 必填 | 说明 |
|------|------|------|
| key | ✅ | API Key |
| city | ✅ | adcode（城市编码），如 `110101` |
| extensions | 可选 | `base`=实况 / `all`=预报天气（4天） |
| output | 可选 | JSON(XML) |

### 3.2 实况天气 (extensions=base)

返回 `lives` 数组：

| 字段 | 类型 | 说明 |
|------|------|------|
| province | string | 省份名 |
| city | string | 城市名 |
| adcode | string | 区域编码 |
| weather | string | 天气现象（汉字，如"晴"、"小雨"） |
| temperature | string | 实时气温（℃） |
| winddirection | string | 风向描述（如"东北"） |
| windpower | string | 风力级别（如"≤3"、"5"） |
| humidity | string | 空气湿度 |
| reporttime | string | 数据发布时间 |

### 3.3 预报天气 (extensions=all)

返回 `forecasts[].casts[]`（当天+未来3天，共4天）：

| 字段 | 说明 |
|------|------|
| date | 日期 |
| week | 星期几 |
| dayweather / nightweather | 白天/夜间天气现象 |
| daytemp / nighttemp | 白天/夜间温度 |
| daywind / nightwind | 白天/夜间风向 |
| daypower / nightpower | 白天/夜间风力 |

### 3.4 更新频率
- **实况**：每小时更新多次
- **预报**：每天 8:00、11:00、18:00 左右各更新一次

### 3.5 ⚠️ 功能缺失
| 缺失项 | 影响 |
|--------|------|
| 无小时级预报 | 无法做 24h 温度曲线 |
| 无降水量（mm） | 无法做雨量反馈 |
| 无 AQI / 空气质量 | 无法做空气质量卡片 |
| 无降水概率 | 无法做降雨概率 |
| 无体感温度 | 无法做体感对比 |
| 无日出日落 | 无天文数据 |

> **结论：高德天气 API 是基础兜底源，只能提供温度+风+4天预报+天气描述。需要和风/彩云补充细节。**

## 4. 行政区查询 API `/v3/config/district`

### 4.1 请求参数

| 参数 | 必填 | 说明 |
|------|------|------|
| key | ✅ | API Key |
| keywords | 可选 | 行政区名称 / citycode / adcode |
| subdistrict | 可选 | 子级行政区级数：0|1|2|3 |
| extensions | 可选 | `base`(无边界) / `all`(返回边界 polyline) |
| filter | 可选 | 按 adcode 过滤，只返回该省/直辖市信息 |
| offset | 可选 | 最外层返回数据个数(默认20) |
| page | 可选 | 页码 |

### 4.2 返回结构（关键字段）

```json
{
  "districts": [{
    "adcode": "220100",        // 区域编码
    "name": "长春市",
    "level": "city",
    "center": "125.3245,43.8868",
    "polyline": "125.32,43.88;125.33,43.89;...",  // 边界坐标（extensions=all时）
    "districts": [{            // subdistrict>=1 时返回子级
      "adcode": "220102",
      "name": "南关区",
      "level": "district",
      "center": "125.3508,43.8640",
      "polyline": "..."
    }, ...]
  }]
}
```

### 4.3 与用户场景的契合度

**完美契合**「周边区县天气地图」：
1. `keywords=长春市&subdistrict=1&extensions=all` → 一次性返回所有区县 + 边界 polyline
2. 每个区县拿到 adcode → 逐区调天气 API
3. Android Map SDK 用 polyline 绘制区县边界 → 填充天气颜色/图标

```
流程：GPS定位 → 逆地理编码(得adcode+区名)
     → 行政区API(subdistrict=1) → 全市所有区县列表+边界
     → 逐区天气API → 各区实况
     → Android地图渲染：polyline边界+天气覆盖层
```

## 5. 逆地理编码 API `/v3/geocode/regeo`

### 5.1 请求参数

| 参数 | 必填 | 说明 |
|------|------|------|
| key | ✅ | API Key |
| location | ✅ | 经纬度 `lng,lat`，小数点不超6位 |
| extensions | 可选 | `base`(基本地址) / `all`(含POI+道路) |
| radius | 可选 | 附近 POI 搜索半径 0-3000m(默认1000) |

### 5.2 返回关键字段

| 字段 | 说明 |
|------|------|
| regeocode.addressComponent.province | 省 |
| regeocode.addressComponent.city | 市（直辖市为空） |
| regeocode.addressComponent.district | 区县 |
| regeocode.addressComponent.adcode | 行政区编码（如 220102） |
| regeocode.addressComponent.township | 乡镇/街道 |
| regeocode.addressComponent.towncode | 街道编码 |

> adcode 可直接喂给天气 API + 行政区 API

## 9. Android 地图 SDK（用户天气地图设想的技术基础）

用户的想法：**以区级行政区为粒度，展示本区及相邻区县天气的 Android 地图界面。**

### 9.1 核心能力

| SDK 模块 | 能力 | 天气地图中的应用 |
|----------|------|----------------|
| MapView 地图显示 | 3D地图渲染、缩放、旋转 | 底图承载 |
| Polygon 多边形覆盖物 | 绘制填充区域 | 区县边界 + 天气底色（雨=蓝、晴=黄） |
| Marker 标记 | 图标+文字标记 | 各区温度标签 |
| TileOverlay 瓦片覆盖 | 自定义图层 | 雷达图叠加 |
| CameraUpdate 视角控制 | 自动缩放到最佳视野 | 聚焦当前城市 |

### 9.2 技术实现路径

```
Web API 层（数据获取）:
  GPS → 逆地理编码 → adcode + 区名
  → 行政区 API(subdistrict=1) → 所有区县边界+adcode
  → 批量天气 API → 各区实况/预报

Android SDK 层（地图渲染）:
  → MapView 显示
  → Polygon 绘制各区边界（fills = 天气颜色映射）
  → Marker 标注各区温度/天气图标
  → 自定义 InfoWindow 点击展开详情
```
